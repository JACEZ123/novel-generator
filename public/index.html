流程说明

0 · 一次性配置

点右上角「🔑 设置」，填入你的模型 API Key（默认对接 DeepSeek，也可改 Base URL 接任意 OpenAI 兼容接口）。密钥只保存在你本机的 data/config.json，不会上传，也不会随仓库提交。也可用环境变量 NOVEL_API_KEY 注入。

1 · 建书 + 生成设定

填写书名、题材、目标章数与初始设定（世界观、金手指、主角、风格）。系统据此生成故事框架、分卷卷纲、角色卡、伏笔与文风，可逐项编辑或给意见重生成。

2 · 章纲规划 + 结构审计

每 5 章为一组生成章纲（标题 + 一句话 + 梗概），并自动做结构审计：检查是否服务总纲/分卷目标、前后承接、读者心智，不合格自动按意见修订。你可逐章编辑或整体给意见。

3 · 逐章写作 + 连贯性审计

按已确认章纲写正文，写完自动做连贯性/设定一致性/节奏审计，发现问题按等级自动修订，直到通过。手动模式下每章可人工确认、重写或改写。

4 · 人物面板（游戏/系统流可选）

游戏/LitRPG 题材会维护主角等级/属性/装备/技能面板：每章把当前面板作为上下文注入，写完依据正文自动更新数值，每 5 章在正文中完整展示一次。

5 · 自动连写 + 断点续写

可一键「自动写到完本」：后台持续写作，关闭页面也不中断；随时刷新即可看到最新进度与正文。配合看门狗脚本可自愈重启。

模型配置

不同写作阶段可用不同模型：大纲/审计/修订用更强的模型（质量更高），正文/审查用更快的模型（更省更快）。可用模型列表在「模型服务」里维护。

Skills 技能管理

全部技能都展示在这里，均可查看与修改（存于 data/skills/）。写作流程（生效）组是本应用真实使用的流程提示词，改完保存立即生效；引擎能力（改写版）组是引擎内置能力技能的自有改写版，供查看/复用；自定义组放你自己的片段。内置技能删除后会自动恢复默认。
